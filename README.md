# GovCon Weekly Intelligence

Weekly intelligence newsletter for government contractors. Delivers 70-80% of the value of $5K-$200K/yr enterprise platforms (GovWin, BGOV, Govini) at $249-$699/yr by cross-referencing free public data sources with AI synthesis and editorial voice.

## Quick Start

```bash
cd ~/Personal/govcon-intel

# Full pipeline: pull data → AI enrich → generate newsletters
./generate.sh

# Or run each step individually:
python3 pipeline.py               # 1. Pull raw awards from USAspending API
python3 enrich.py                 # 2. AI enrichment via AWS Bedrock (Claude Sonnet)
python3 generate_newsletter.py    # 3. HTML email newsletter
python3 generate_substack.py      # 4. Substack-ready markdown
```

## Two Output Formats

| Generator | Output | Format | Where to use |
|-----------|--------|--------|-------------|
| `generate_newsletter.py` | `output/report_*_v2.html` | Styled HTML (18KB) | Email sends (Beehiiv, ConvertKit, direct) |
| `generate_substack.py` | `output/substack_*.md` | Clean markdown (11KB) | Paste into Substack editor |

**Why two?** Substack strips all custom CSS. The HTML version has styled ticker tables, gold category labels, dark header/footer. The Substack version uses only what Substack supports: headers, bold, lists, pipe tables, emoji dots for color, horizontal rules.

Both read from the same source: `data/corrected_all.json` (generated by `enrich.py`)

## Design

Morning Brew style — not a dashboard or AI report. Key principles:
- Helvetica/Arial, white background, no cards/badges/colored borders
- Bold lede sentence → short follow-up paragraph
- Bullet lists with bold lead-ins
- Simple data tables (ticker-style)
- Gold (#C9A227) category labels in caps
- Section dividers with `<hr>`
- Content carries everything, not design chrome

## Data Model

Source file: `data/corrected_all.json`

```
{
  report_date, report_title, report_subtitle,
  notable           → headline + key stat (lede)
  doge_tracker      → agencies_affected, court_rulings, contractor_impact
  sections/
    recompete_alerts  → items[] with wired_score, winnability, who_should_pursue
    new_awards        → items[] with why_they_won, market_signal, what_losers_should_do
    option_exercises  → items[] with recompete_signal, who_should_watch
    funding_actions   → items[] with intel_note, new_ceiling
  market_pulse      → total_obligations, top_agencies, trending_naics
  action_items      → priority-ranked with deadlines
  one_to_watch      → forward-looking program intelligence
  govcon_health_index → 0-100 composite score
  is_it_wired       → items[] with wired_score, signals, verdict
  bridge_watch      → items[] with bridge_period, what_to_do
  set_aside_spotlight → underperforming_agencies[]
  protest_report    → notable_decisions[] with lesson
  calendar          → this_week, next_two_weeks, looking_ahead
}
```

## Newsletter Sections

1. **Procurement Pulse** — ticker table: obligations, agency spend, health index, bidder density
2. **DOGE Watch** — agency-by-agency impact bullets + court rulings
3. **Who Won This Week** — awards with bold lede + "why they won" + market signal
4. **Is It Wired?** — 0-100 score table on recompetes + editorial picks (chase/skip/toss-up)
5. **Bridge Watch** — contracts on life support = imminent recompetes
6. **Quick Hits** — funding actions + option exercises, one-liner format
7. **Small Business** — set-aside gap table by agency/category
8. **Protest Corner** — outcome + tactical lesson per protest
9. **Your To-Do List** — numbered actions with deadlines + calendar
10. **One More Thing** — forward-looking program intelligence

## External Dependencies

- **USAspending API** — Free, no key. Awards, mods, spending data.
- **AWS Bedrock** — Claude Sonnet for AI enrichment. Requires `boto3` + AWS SSO credentials.
- **SAM.gov API** — Free key. Solicitations, entities. (planned)
- **Python 3** + `boto3` (for Bedrock). All other code uses stdlib only.
